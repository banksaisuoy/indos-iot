# 🦆 คู่มือการใช้งาน AeroDuck Pro v19.0-RS485 (Farm Controller)

คู่มือฉบับนี้ใช้สำหรับบอร์ด ESP32 ควบคุมพัดลมโรงเรือนเป็ดไข่ (ระบบปิด) เวอร์ชัน 19.0-RS485 ซึ่งเปลี่ยนมาใช้เซ็นเซอร์ SHT20 ผ่าน RS485 เพื่อความเสถียร ใช้งานต่อเนื่อง 24/7 และปลอดภัยสำหรับเป็ด

---

## 1. การตั้งค่าและใช้งานผ่าน Web Dashboard (Web UI)

เข้าสู่ระบบผ่านเบราว์เซอร์ด้วย IP Address ของบอร์ด (เช่น `http://192.168.1.50`)

### 📊 หน้าจอ Dashboard แบบ Real-time
*   **Top Bar**: แสดงสถานะสัญญาณ WiFi, สถานะการเชื่อมต่อ MQTT (Node-RED) และ Uptime
*   **THI (Poultry Stress Index)**: ดัชนีความเครียดของเป็ด คาดเดาจากอุณหภูมิและความชื้น 
    *   *Comfortable (สีเขียว)*: เป็ดสบายตัว กินอาหารได้ปกติ
    *   *Mild Stress (สีเหลือง)*: เริ่มเครียดเล็กน้อย อาจกินน้อยลง
    *   *Severe Stress (สีส้ม)*: เครียดสะสม ระวังไข่ลด
    *   *DANGER! (สีแดง)*: เสี่ยงตายจาก Heat Stress (ฮีทสโตรก) พัดลมจะเปิดทั้งหมด
*   **Temperature & Humidity**: กราฟเส้นแบบ Sparkline แสดงแนวโน้มอุณหภูมิและความชื้น
*   **Fan G1 / G2 Status**: แสดงสถานะเปิด/ปิด และเหตุผลที่ทำงาน (เช่น Running (Target Met), Cooldown, OFF)

### 🎛️ การสั่งงาน Manual (Manual Override)
เมื่อต้องการบังคับเปิดพัดลมตอนทำความสะอาด หรือต้องการไล่อากาศ:
1. เลื่อน Slider เพื่อเลือกเวลา **Manual Timer** (1 ถึง 120 นาที)
2. กดปุ่ม `▶️ Start Manual` ไอคอนพัดลมจะสว่างทันที และมีแถบ Progress Bar วิ่งถอยหลัง
3. หากต้องการยกเลิกกลางคัน ให้กดปุ่ม `🛑 Stop (Auto)` ระบบจะกลับสู่โหมดอัตโนมัติ

## 2. การจำลองปัญหาหน้างานจริง (Troubleshooting & Real-world Scenarios)

บอร์ดเวอร์ชันนี้ถูกออกแบบมาให้แก้ไขปัญหาเฉพาะหน้าได้เองโดยไม่ทำให้เป็ดตาย (Fail-safe mechanism)

### ⚠️ Scenario A: เซ็นเซอร์พัง, สายหลุด, เสื่อมสภาพ (RS485 SHT20 Failure)
*   **อาการ**: หน้าจอขึ้นคำว่า `FAILSAFE!` และอุณหภูมิแสดง `--.-°C`
*   **การทำงานของระบบ**: 
    1. เซ็นเซอร์อ่านค่าไม่ได้ 3 ครั้งติด ระบบจะสั่งเปิดพัดลม **G1 และ G2 พร้อมกัน (100%)** เพื่อรับประกันว่าอากาศถ่ายเท (ปลอดภัยไว้ก่อน)
    2. เสียงไซเรนเตือนแบบนุ่มนวล (Soft Siren: ความถี่ต่ำ 400Hz/600Hz) จะดังเป็นจังหวะ เพื่อเรียกให้คนงานมาดู โดย**ปรับเป็นโทนเสียงต่ำ เพื่อลดความแหลมของเสียงและลดโอกาสรบกวนสัตว์ในโรงเรือน**
*   **วิธีแก้**: ตรวจสอบสายไฟเซ็นเซอร์ RS485 (สาย A, B หรือสายไฟเลี้ยง) เสียบใหม่ หรือเปลี่ยนเซ็นเซอร์ หากเสียบกลับเข้าไปแล้ว ระบบจะอ่านค่า Modbus ใหม่และกลับมาทำงาน Auto ทันที

### 🔌 ข้อมูลการต่อสายไฟฮาร์ดแวร์ (Hardware Wiring - ESP32-S3)
*   **RS485 RX (RO)** -> `GPIO 16`
*   **RS485 TX (DI)** -> `GPIO 17`
*   **RS485 DE/RE** -> `GPIO 4` (เชื่อมต่อ DE และ RE เข้าด้วยกันแล้วต่อเข้า GPIO 4)
*   **Relay G1** -> `GPIO 18` (ทำงานเมื่อสถานะเป็น LOW)
*   **Relay G2** -> `GPIO 19` (ทำงานเมื่อสถานะเป็น LOW)
*   *ข้อควรระวัง*: รีเลย์ของบอร์ด ESP32 ออกแบบมาเพื่อขับคอยล์ของแมกเนติกคอนแทคเตอร์ (Contactor coil) เท่านั้น **ห้ามนำไปขับโหลดมอเตอร์พัดลมโดยตรงเด็ดขาด**

### 🌐 Scenario B: เน็ตหลุด, เราเตอร์ค้าง (WiFi Drop)
*   **อาการ**: ไอคอน WiFi ด้านบนกระพริบ หรือหน้าเว็บหมุนค้าง
*   **การทำงานของระบบ**: 
    *   หากเชื่อมต่อ WiFi เดิมไม่ได้ บอร์ดจะปล่อย Hotspot ออกมาชื่อ **`DuckFarm-Setup`** (รหัสผ่าน `12345678`)
    *   **หมายเหตุ**: ระหว่างที่ไม่มี WiFi ระบบควบคุมอุณหภูมิ (Auto) และ Failsafe ยังคง**ทำงานปกติ 100%** เพียงแค่จะส่งข้อมูลเข้า Grafana ไม่ได้
*   **วิธีแก้**: ใช้มือถือเชื่อมต่อ WiFi `DuckFarm-Setup` เข้าเว็บ `192.168.4.1` แล้วกดรูปฟันเฟืองเพื่อใส่รหัสผ่าน WiFi ของฟาร์มใหม่ 

### ⚡ Scenario C: ไฟตก, ไฟกระชาก, บอร์ดแฮงก์ (Power Surge / Reboot)
*   **อาการ**: พัดลมดับไปชั่วขณะ และไฟที่บอร์ดดับ
*   **การทำงานของระบบ**:
    1. เมื่อไฟมา บอร์ดจะใช้เวลาวอร์มเซ็นเซอร์ 20 วินาที (ขึ้นสถานะ `WARMUP`)
    2. ระหว่างวอร์มอัพ พัดลมจะยังไม่หมุน เพื่อป้องกันไฟกระชาก (Surge) ที่อาจทำให้เบรกเกอร์ทริป
    3. หลัง 20 วินาที จะอ่านค่าและสั่งงานพัดลมทันที

### ⏳ Scenario D: มอเตอร์ต้องการพัก (Min Run & Cooldown Protection)
*   **อาการ**: หน้าจอขึ้นสถานะอย่างใดอย่างหนึ่ง แต่พัดลมอาจทำงานขัดกับความรู้สึก
*   **การทำงานของระบบ**: 
    *   **Min Run (ระบบบังคับทำงานต่อ):** เมื่อพัดลมเพิ่งเริ่มทำงาน ระบบจะบังคับให้เปิดต่อเนื่องตามเวลาขั้นต่ำ (Min Run) ที่ตั้งไว้ เพื่อป้องกันการเปิดๆ ปิดๆ ถี่เกินไป (รักษาระดับกระแสไฟ)
    *   **Cooldown / Anti short-cycle (ระบบหน่วงเวลาพัก):** หลังจากพัดลมเพิ่งดับลง ระบบจะหน่วงเวลาพักก่อนอนุญาตให้มอเตอร์เริ่มสตาร์ทใหม่ (แม้ว่าอุณหภูมิจะเกินแล้วก็ตาม) เพื่อถนอมมอเตอร์และลดกระแสไฟกระชาก
*   **วิธีแก้**: รอจนเวลาในหน้าจอนับถอยหลังหมด พัดลมจะกลับมาทำงานตาม Auto ปกติ ห้ามดึงเบรกเกอร์บายพาสระบบเพราะอาจทำให้พัดลมไหม้ได้

## 3. ขั้นตอนการนำไปติดตั้ง และ Flash Firmware (Deployment)

### 3.1 การ Flash สาย USB (ครั้งแรกที่ออฟฟิศ)
1. ติดตั้ง Arduino IDE
2. นำไฟล์ `duck_farm_v19_Relay6_rs485.ino` ไปเปิด
3. เลือกบอร์ด: **ESP32S3 Dev Module**
4. ตั้งค่า Partition Scheme: **16M Flash (3MB APP / 9.9MB FATFS)** -> *สำคัญมากสำหรับ OTA*
5. ตั้งค่า PSRAM: **OPI PSRAM**
6. กด Upload 

### 3.2 การทำ OTA (อัปเดตระยะไกลหน้าฟาร์ม)
เมื่อติดตั้งบอร์ดที่กล่องคอนโทรลแล้ว ไม่ต้องเปิดตู้เพื่อเสียบสายอีกต่อไป:
1. ใน Arduino IDE ไปที่เมนู `Sketch` -> `Export Compiled Binary`
2. จะได้ไฟล์ `.bin` (เช่น `duck_farm_v19_Relay6_rs485.ino.bin`)
3. เปิดหน้าเว็บของบอร์ด (เช่น `http://192.168.1.50/update`)
4. อัปโหลดไฟล์ `.bin` รอจนเสร็จ 100% บอร์ดจะรีบูตตัวเอง และพร้อมใช้งาน Firmware ใหม่ทันที

## 4. Hardware Fail-safe Checklist (สำคัญมากสำหรับการทำ POC/UAT)
เพื่อความปลอดภัยสูงสุดและป้องกันเป็ดตายจากความร้อนสะสม ระบบซอฟต์แวร์ไม่สามารถป้องกันความเสี่ยงได้ 100% จึงต้องมีระบบฮาร์ดแวร์สำรองเสมอ:
- [ ] ติดตั้ง Thermostat อนาล็อก (ตั้งไว้ที่ 35°C) ต่อสั่งแมกเนติกคอนแทคเตอร์โดยตรง เพื่อให้พัดลมทำงานอัตโนมัติหากบอร์ด ESP32 ค้าง
- [ ] มีสวิตช์ Selector Manual Override (Auto-Off-Manual) ที่ตู้คอนโทรล เพื่อให้คนงานบิดเปิดพัดลมได้โดยตรง
- [ ] เบรกเกอร์และฟิวส์สำหรับมอเตอร์พัดลมแยกจากวงจรไฟเลี้ยงบอร์ด ESP32 เพื่อป้องกันไฟกระชากจากมอเตอร์ทำให้บอร์ดรีบูต

## 5. แผนการพัฒนาในอนาคต (v19 TODOs)
- **Logging System**: เปลี่ยนการเขียน log ไปใช้ Queue-based LittleFS (Log Worker Task) เพื่อหลีกเลี่ยง blocking I/O
- **Sensor Redundancy**: รองรับเซ็นเซอร์ 2-3 ตัว พร้อมระบบหาค่าเฉลี่ยและตัดตัวที่เสียทิ้ง (Outlier rejection)
- **Hardware Watchdog**: เพิ่มวงจร Watchdog ภายนอก (IC/Timer Relay) เพื่อตัดต่อไฟรีเซ็ตบอร์ดหากซอฟต์แวร์แฮงก์สนิท
